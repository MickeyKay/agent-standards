# Product Opportunity Assessment

## Gather First

- target user segment and unmet need
- current user behavior, friction, and alternatives
- strategic goal (growth, retention, revenue, efficiency, quality)
- known signals: qualitative feedback, usage data, support themes, competitive context
- constraints: capacity, timeline, compliance, platform dependencies

## Workflow

1. Define the core user problem and why it matters now.
2. Clarify the opportunity mechanism: how solving this problem creates measurable value.
3. Estimate impact potential and confidence level from available evidence.
4. Identify key uncertainties and what would disconfirm the idea.
5. Compare 2–3 realistic option shapes (including a minimal path and a no-build path).
6. Recommend a direction with explicit tradeoffs and expected downside.
7. Propose the smallest experiment or MVP slice to validate assumptions quickly.

## Output Expectations

- opportunity summary (problem, user, value hypothesis)
- options and recommendation
- confidence level with evidence quality
- next-step experiment plan and success/failure signals

## Completion Criteria

- recommendation is explicit and decision-ready
- key assumptions and risks are visible
- next step is small enough to execute quickly

## Anti-Patterns

- treating feature enthusiasm as evidence of opportunity
- proposing an MVP too large to validate quickly
- skipping no-build or defer alternatives
